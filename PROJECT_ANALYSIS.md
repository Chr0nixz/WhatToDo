# WhatToDo 项目分析与下一步开发计划

更新时间：2026-05-31（Asia/Shanghai）

## 1. 项目现状

WhatToDo 是一个本地优先的桌面任务与 DDL 管理应用，技术栈为 Tauri 2、React 19、TypeScript、Vite、Tailwind CSS 4 和 SQLite。产品方向保持清晰：围绕每日 DDL、项目任务、工作区、工作文件夹、系统提醒、提醒中心、恢复中心、导入导出、保存视图和重复任务，降低桌面工作流里的切换成本。

当前代码结构：

- `src/data`：类型定义、日期工具、提醒中心分组、项目计算、任务筛选、自然语言快速添加、重复任务、repository 抽象与 Local/SQLite 实现。
- `src/hooks`：任务数据、主题、提醒 tick 等应用级逻辑。
- `src/components/app`：主页、总览、日期面板、任务详情、项目、工作区、提醒中心、恢复/设置、更新设置、悬浮窗等主要 UI。
- `src-tauri`：SQLite 迁移、系统托盘、悬浮窗、文件读写命令、Tauri 插件和更新器集成。

当前自动化基线健康：

- `pnpm test` 通过：12 个测试文件，43 个用例。
- `pnpm build` 通过。
- `cargo check` 通过。
- `git diff --check` 无空白错误，仅有 Windows CRLF 提示。

构建中仍有一个性能提示：前端主 JS chunk 约 530.88 kB，超过 Vite 500 kB 警告阈值，后续需要代码拆分。

## 2. 已完成能力

### 2.1 任务、项目和工作区

- 每日 DDL 主页，支持月历/周视图、选中日期任务列表、逾期提示和搜索。
- 总览页支持任务搜索、状态范围、优先级、项目、提醒、文件夹、日期范围筛选。
- 总览页支持保存视图、应用保存视图和删除保存视图。
- 项目页支持项目创建、项目截止日、项目工作文件夹、归档项目、按项目查看任务。
- 工作区页支持工作区创建、工作区切换、常用文件夹、从其他工作区加入已有任务、工作区悬浮窗。
- 任务详情页支持标题、备注、日期、时间、优先级、项目、任务文件夹、完成状态和删除。

### 2.2 提醒与提醒中心

- 提醒模型包含 `remindAt`、`snoozedUntil`、`firedAt`、`failedAt`、`lastError`、`lastAttemptedAt` 和 `enabled`。
- 到期提醒筛选排除已禁用、已触发、已失败、已完成任务、已删除任务和未来提醒。
- `useReminders` 一次 tick 会处理所有已到期提醒。
- 提醒失败会被持久化，并在提醒中心的失败分组中展示。
- 提醒中心按失败、已错过、即将提醒、已触发分组展示。
- 提醒中心支持打开任务、完成任务、重试失败提醒、稍后提醒和关闭提醒。
- 稍后提醒固定三项：10 分钟、1 小时、明天 09:00 本地时间。

### 2.3 恢复、导入导出和数据安全

- 删除任务、删除工作文件夹和归档项目已经有 undo toast。
- 设置页有恢复中心，可恢复已删除任务、已删除工作文件夹和已归档项目。
- 支持 JSON 备份导出和导入。
- 导入备份前会先生成 pre-import 备份文件，降低覆盖风险。
- 支持当前工作区任务导出为 CSV 和 ICS。
- Backup payload 已升级到版本 2，包含重复任务模板；版本 1 备份仍可导入。

### 2.4 重复任务和快速添加

- 支持创建 daily、weekly、monthly 重复任务。
- 创建重复任务时会创建模板和首个任务实例。
- 完成当前重复实例时会生成下一次实例，并避免重复生成相同日期实例。
- 支持关闭重复模板，关闭后不再生成未来实例。
- 支持快速添加解析，例如：

```text
明天下午3点交周报 #工作 !高 提前30分钟
```

解析结果包括标题、日期、时间、项目、优先级和提醒提前量。

### 2.5 i18n、主题和桌面运行

- 中英文 UI 文案集中在 `src/i18n/index.ts`，中英文 key 数量一致。
- 日期标题、月份、周几、提醒中心时间已按当前语言本地化。
- 支持 light、dark、system 主题，以及蓝、绿、黄、玫瑰、紫色主题色。
- Tauri dev 使用 `scripts/tauri-before-dev.mjs` 复用或启动 Vite dev server。
- Tauri 托盘、关闭到托盘、悬浮窗、窗口状态、更新器、通知、SQL、dialog、opener 已接入。

## 3. 四个维度的当前不足与改进点

### 3.1 用户交互便捷性

当前体验已经能完成日常任务流，但还有几个高价值摩擦点：

- 提醒编辑不够直接。创建任务时只有提醒开关，提醒提前量主要来自默认设置或快速解析；任务详情只展示提醒时间，不能直接调整提醒提前量、重新开启提醒或改为自定义提醒。
- 删除/归档仍先弹 `window.confirm`，然后再提供 undo。恢复能力已经存在，常规可恢复操作可以改成直接执行并显示撤销，只对不可恢复操作保留确认。
- 快速添加解析缺少预览。用户点击解析后只看到表单被套用，无法确认哪些 token 被识别为日期、时间、项目、优先级或提醒。
- 缺少桌面效率入口。全局快捷键快速新增、命令面板、快速打开文件夹、快速切换工作区还未实现。
- 打开文件夹、导入导出、更新检查等系统操作失败时反馈偏弱，部分 `openPath` 调用失败后没有局部错误提示。
- 日期展示还不完全统一。部分任务行仍显示原始 `YYYY-MM-DD`，详情提醒使用系统 `toLocaleString()`，与中英文日期格式不完全一致。
- 一些 icon-only 按钮依赖 `title`，还可以补显式 `aria-label`，让键盘和读屏体验更稳。

建议方向：

1. 给任务创建和详情页增加提醒提前量选择器：无提醒、10 分钟、30 分钟、1 小时、1 天、自定义。
2. 把任务删除、文件夹删除、项目归档改成非阻塞 undo 流。
3. 为快速添加增加解析结果 chips，并支持一键清除解析结果。
4. 增加命令面板和全局快捷键，优先覆盖新增任务、搜索任务、打开文件夹、切换工作区。
5. 统一日期展示走 `src/data/dateFormat.ts`。

### 3.2 程序功能丰富性和完整性

当前功能面已经从基础任务管理扩展到提醒失败、恢复、备份、保存视图和重复任务，但不少能力仍是初版：

- 重复任务规则偏基础。现在只支持每天、每周、每月，`interval` 实际固定为 1，没有每 2 周、每周一三五、每月最后一天、结束次数等规则。
- “更新未来重复”目前主要更新模板，不会同步已经生成的未来实例，按钮文案容易让用户误以为所有未来任务都已更新。
- 提醒历史还不完整。当前保存最后失败原因和尝试时间，但没有完整事件日志记录触发、关闭、稍后、重试、成功的时间线。
- 保存视图有创建/删除/应用，但缺少重命名、覆盖当前视图、设为默认、排序或置顶。
- 项目和工作区管理还不完整。项目缺少编辑名称、颜色、截止日、删除入口；工作区缺少编辑和删除入口。
- 数据导入导出还缺少自动定期备份、导入预览、选择性合并、schema 细粒度校验。
- ICS 导出偏基础，缺少时区、提醒 `VALARM` 和更准确的结束时间。
- 快速添加解析规则仍偏窄，对“下周三”“周五前”“月底”“每周一”“不提醒但保留日期”等自然表达支持不足。

建议方向：

1. 将重复任务拆成明确的 recurrence rule 模型，并区分“更新模板”“更新未完成未来实例”“只更新当前实例”。
2. 增加提醒事件日志表或事件数组，提醒中心可查看历史。
3. 补齐保存视图管理能力：重命名、覆盖、默认视图、排序。
4. 补齐项目和工作区编辑/删除能力。
5. 增强备份导入流程：校验、预览、合并策略、自动备份计划。

### 3.3 项目架构的鲁棒性和稳定性

当前测试和构建通过，但真实桌面运行、并发提醒、SQL 原子性和安全边界仍需要加固：

- `useReminders` 存在潜在重复触发竞态。effect 依赖整个 `data`，每次标记提醒触发或失败都会重建 effect；旧 tick 没有取消令牌，新 tick 会立即开始，多个提醒同时到期时可能出现重复处理。
- SQLite 多步写入缺少事务。创建任务和提醒、创建重复模板和首个实例、更新任务并更新提醒、完成重复任务并生成下一实例，任一步失败都可能留下半完成状态。
- SqlRepository 测试覆盖偏浅。当前 LocalRepository 覆盖较多，SQLite 侧主要验证了 snooze/disable，重复任务、备份导入、保存视图、失败提醒、workspace 过滤和软删除恢复仍需要 SQL 语义测试。
- LocalRepository 与 SqlRepository 仍有语义差异风险。SQLite 是每 workspace 一份 settings；Local fallback 当前是单份 settings，无法完全模拟多工作区设置行为。
- Tauri 安全边界需要收紧。`tauri.conf.json` 的 CSP 仍为 `null`，capability 作用于 `windows: ["*"]`，并暴露了任意路径 `read_text_file` 和 `write_text_file` 命令。
- `open_workspace_window` 参数需要更严格校验和 URL 编码，避免导入数据中的特殊 workspace id 或 title 影响窗口 label、URL 和日志。
- Tauri 实机验证仍未闭环。通知权限允许/拒绝、托盘恢复、关闭到托盘、悬浮窗置顶/折叠/尺寸、真实文件夹打开都需要在 `pnpm tauri dev` 下逐项确认。

建议方向：

1. 重构 `useReminders`：使用稳定 ref 保存最新 data/action，给 tick 增加运行锁和取消令牌，并按提醒 id 做本轮去重。
2. 为 SQLite 多步 mutation 增加事务。
3. 扩展 SqlRepository 测试，覆盖 LocalRepository 已覆盖的关键语义。
4. 将 LocalRepository 的 settings 改成 workspace 维度，或明确限制 browser fallback 的语义范围。
5. 收紧 CSP、capability windows 范围、命令参数校验和文件读写路径范围。
6. 建立桌面实机验证清单，并在每次发布前执行。

### 3.4 程序运行效率

当前小数据量体验应当可靠，但数据规模上来后会遇到查询、渲染和 bundle 体积问题：

- SqlRepository 每次 mutation 后基本都会 `readAll()`，一次读取当前 workspace 的 projects、tasks、availableTasks、folders、deleted items、reminders、settings、saved views 和 recurring templates。
- 多处查询使用 `SELECT *`，会把当前视图暂时不需要的字段也拉出来。
- `availableTasks` 每次 `readAll()` 都查询所有其他 workspace 的任务，即使当前界面并不需要。
- Overview、Home、WorkspaceTaskPicker、悬浮窗都在前端完整 filter/sort/render，任务上千后会明显变慢。
- 搜索没有 debounce，Overview 每次输入都会过滤、查项目、排序。
- `dueRemindersForData` 中 `reminders.filter` 内部反复 `tasks.find`，提醒多时是 O(n*m)。
- 前端构建出现 chunk 体积警告，主 JS chunk 约 530.88 kB。

建议方向：

1. 将 repository 拆出按视图查询能力：今日任务、总览分页、提醒中心、项目任务、工作区任务。
2. `availableTasks` 改为打开任务选择器时按需加载。
3. Overview 和 WorkspaceTaskPicker 增加分页或虚拟滚动。
4. 搜索输入增加 debounce。
5. 提醒筛选先建 `tasksById` Map。
6. 对设置、更新器、导入导出、Radix dialog、date-fns locale 或 Tauri 插件做动态 import 和代码拆分。

2. 为 SQLite 多步写入增加事务。
3. 收紧 Tauri CSP、capability 和文件读写命令边界。

4. 修正“更新未来重复”的真实语义，或调整文案避免误导。

1. 增强重复任务规则。
2. 增加提醒历史。

4. 补齐项目和工作区编辑/删除。

## 6. 短期建议

最建议先做：

1. `useReminders` 竞态修复。
2. SQLite 多步写入事务。
3. Tauri 安全边界收紧。
4. Tauri 桌面实机验证。

原因是这四项直接决定“任务管理应用是否可信”：提醒不能重复或漏发，数据不能半写入，桌面权限不能过宽，真实桌面环境要有验证闭环。
