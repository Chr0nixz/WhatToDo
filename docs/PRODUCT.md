# Product

## Register

product

## Users

WhatToDo is for deadline-driven people who work from a desktop: students, developers, operators, and knowledge workers who need a local place to track dates, project work, reminders, and working folders. Users are usually in execution mode, moving between a calendar, a project list, a task detail pane, and the file system.

## Product Purpose

The product is a local-first desktop task manager for deadline work. It combines daily DDL planning, all-task overview, project grouping, task-level reminders, bilingual UI, theme settings, and working-folder shortcuts. Success means a user can see what is due, add a task quickly, open the right project folder, and return to work without context switching.

## Brand Personality

Calm, precise, dependable. The interface should feel like a quiet command center: compact enough for repeated daily use, structured enough to be trusted, and restrained enough that deadlines stay more important than decoration.

## Anti-references

- Marketing-first hero layouts that explain the app instead of letting the user work.
- Decorative gradient text, glassmorphism, neon dashboards, and heavy purple-blue AI-tool styling.
- Overly cute productivity UI with playful mascots, oversized empty states, or motivational copy.
- Spreadsheet-like density without hierarchy.
- Non-standard controls that make common task operations feel invented or unfamiliar.

## Design Principles

1. **Work is the first screen.** The app opens into actionable views: calendar, overview, projects, settings. Avoid landing-page framing inside the product shell.
2. **Dates and status lead.** Due date, completion state, priority, project, and reminder state should be visible without opening a task whenever space allows.
3. **Local context matters.** Working-folder controls are part of the task workflow, not secondary metadata.
4. **Density needs rhythm.** Compact lists and panels are allowed, but use stable spacing, clear borders, and muted surfaces so the UI remains scannable.
5. **Familiar controls win.** Use standard side navigation, segmented controls, buttons, forms, dialogs, and detail panes. Distinction should come from fit and precision, not novelty.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Support light, dark, and system themes. Preserve visible focus states, keyboard-operable controls, reduced-motion preferences, and bilingual Chinese/English labels. Do not rely on color alone for priority or status; pair color with text, icons, position, or shape.
