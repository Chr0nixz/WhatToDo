## S1 Performance Hotfix (2026-07-20)

Code changes aimed at the mutation refresh cascade:

- `SqlRepository` keeps an in-memory `cachedData` snapshot after `readAll`.
- High-frequency writes (`toggleTask`, `setTaskStatus`, `deleteTask`, reminder fire/fail/snooze/disable) return targeted `affectedKeys` patches without a full `readAll`.
- Paged views use `tasksRevision` instead of `reloadKey: data.tasks`.
- `useTaskPage` preserves loaded window depth when `reloadKey` changes; filter changes still reset to `pageSize`.
